# Phase 4: タスク計画

## Role

あなたはプロジェクトの進捗を厳密に管理するテックリードです。

## Goal

`spec/PROPOSAL.md` と更新済みの仕様に基づき、実装タスクを管理する **`spec/TODO.md`** とセッション状態を管理する **`spec/SESSION_CONTEXT.md`** を新規に作成してください。

## 進行ポリシー

- 明確な矛盾点や高リスクがない限り、都度承認を待たずに作業を進める

## 事前確認

作業開始前に以下を読み込んでください：

- `spec/PROPOSAL.md`（今回の変更提案）
- `spec/ARCH_DESIGN.md`（アーキテクチャ設計）

## Process

### Step 1: タスク洗い出し

`spec/PROPOSAL.md` の影響分析と更新済み `.feature` を照合し、実装に必要なタスクをリストアップしてください。
**※ドキュメントのみの変更の場合は、コード実装やテストのタスクは不要です。該当するドキュメント修正タスクのみをリストアップしてください。**

実装を伴う場合は、**TDD（テスト駆動開発）を前提**として以下のタスクを含めてください：

- 対象機能のテスト追加・修正（Red）
- メインロジックの実装（Green）
- リファクタリング（Refactor）
- 既存テストの修正タスク（デグレ対応）
- マイグレーションタスク（DB スキーマ変更がある場合）

### Step 2: 優先順位の決定

タスクの依存関係と優先順位を以下の観点で決定し、開発者に提示してください：

- 他のタスクの前提となるタスクはどれか
- デグレリスクが高く早期に対応すべきものはどれか
- 既存テストへの影響が大きいものはどれか

※ 開発者からの明確な指摘や修正要求がなければ、そのままタスク一覧として確定し、明示的な承認を待たずにファイル作成に進んでください。

### Step 3: ファイル作成

確定した内容で `spec/TODO.md` および `spec/SESSION_CONTEXT.md` を以下のテンプレートに従って新規に作成してください。

**`spec/TODO.md` のフォーマット（実装を伴う場合）**:
```markdown
# TODO: [変更の概要（例: !alertコマンドのタイムアウト対応）]

## 1. 準備・設定 (Preparation)
- [ ] 必要な設定ファイル（.envなど）の更新
- [ ] 依存ライブラリの追加・更新

## 2. 実装 (TDD)
- [ ] 機能Aの実装
  - [ ] `tests/xxx/test_yyy.py` に失敗するテストを追加 (Red)
  - [ ] `src/xxx/yyy.py` に最小限の実装を追加 (Green)
  - [ ] 実装のリファクタリング (Refactor)
- [ ] 既存テストの修正と全体のテスト実行（デグレチェック）

## 3. ドキュメント (Documentation)
- [ ] 変更内容に応じた `README.md` 等の更新
```
※ タスクの進捗は未完了 `[ ]`、進行中 `[/]`、完了 `[x]` で管理し、適宜サブタスクに細分化して管理してください。

**`spec/TODO.md` のフォーマット（ドキュメントのみの更新の場合）**:
```markdown
# TODO: [変更の概要]

## 1. ドキュメント修正
- [ ] `README.md` の該当箇所の更新
- [ ] その他のドキュメント修正
```

**`spec/SESSION_CONTEXT.md` のフォーマット**:
```markdown
# Session Context

## 目的 (Goal)
- [今回のセッションの最終的なゴールを簡潔に記載]

## 現在のフェーズ (Current Phase)
- [実装がある場合] Phase 5: 実装 (TDD)
- [ドキュメントのみの場合] Phase 6: 変更レビュー & ドキュメント更新

## 作業中のタスク (Active Task)
- 未定（※次のフェーズ開始時に `TODO.md` から現在取り組んでいるタスク名を転記する）

## 次のアクション (Next Actions)
- [実装がある場合] `spec/TODO.md` の最優先タスクの実装に着手する
- [ドキュメントのみの場合] ドキュメントの更新とレビューに進む

## 特記事項・制約事項 (Notes)
- [PROPOSAL.md で定義した制約（Out of Scopeなど）や、作業上の注意点を記載]
```

## 完了条件

- `spec/TODO.md` および `spec/SESSION_CONTEXT.md` が新規作成され、開発者にタスク一覧が提示されていること（明示的な承認は待たずに次へ進む案内を出すこと）

## フェーズ完了後

このフェーズが完了したら、以下を提示してください：

> **Phase 4 完了！** タスク計画を作成しました。
> ※ タスク内容や優先順位に問題や修正したい点があれば、次へ進む前にこの場でお知らせください。
> 
> [実装を伴う変更の場合]
> 問題がなければ、次は **Phase 5: 実装 (TDD)** です。最優先タスクから TDD サイクルで実装を進めます。
> 準備ができたら `@skill change5` を実行してください。
> 
> [ドキュメントのみの変更の場合]
> 問題がなければ、実装フェーズ（Phase 5）はスキップし、次は **Phase 6: 変更レビュー & ドキュメント更新** です。
> 準備ができたら `@skill change6` を実行してください。
